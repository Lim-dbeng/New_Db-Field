# 권한 매트릭스

## 전역 권한별 기본 권한

| 기능 | Super User | Common User | Guest |
|------|-----------|-------------|-------|
| 프로젝트 생성 | ✅ | ❌ | ❌ |
| 프로젝트 삭제 | ✅ | ❌ | ❌ |
| 모든 프로젝트 조회 | ✅ | ✅ (읽기만) | ❌ |
| 프로젝트 멤버 관리 | ✅ (모든 프로젝트) | ⚠️ (본인 프로젝트만) | ❌ |
| 시설물 정보 조회 | ✅ (모든 프로젝트) | ✅ (모든 프로젝트) | ⚠️ (본인 프로젝트만) |
| 시설물 추가 | ✅ (모든 프로젝트) | ⚠️ (본인 프로젝트만) | ⚠️ (본인 프로젝트만) |
| 시설물 수정/삭제 | ✅ (모든 프로젝트) | ⚠️ (본인 생성만) | ⚠️ (본인 생성만) |
| SHP 업로드/관리 | ✅ (모든 프로젝트) | ⚠️ (본인 프로젝트만) | ⚠️ (본인 프로젝트만) |

## 프로젝트별 역할 권한

| 기능 | OWNER | PM | MEMBER |
|------|-------|----|----|
| 프로젝트 설정 변경 | ✅ | ❌ | ❌ |
| 멤버 초대/제거 | ✅ | ⚠️ (MEMBER만) | ❌ |
| 역할 변경 | ✅ | ❌ | ❌ |
| 시설물 승인/반려 | ✅ | ✅ | ❌ |
| 시설물 수정/삭제 | ✅ (모든 시설물) | ✅ (모든 시설물) | ⚠️ (본인 생성만) |
| SHP 레이어 관리 | ✅ | ✅ | ⚠️ (본인 업로드만) |

## 권한 체크 우선순위

1. **Super User**: 모든 권한 자동 허용
2. **프로젝트 멤버십 확인**: `project_members` 테이블에서 역할 확인
3. **소유권 확인**: 본인이 생성한 데이터인지 확인
4. **Guest 제한**: Guest는 프로젝트 멤버십이 있어야만 접근 가능

## 권한 체크 예시

### 시설물 조회
```sql
-- Super User면 모든 시설물 조회 가능
-- Common User/Guest면:
-- 1. 프로젝트 멤버인지 확인
-- 2. 프로젝트 멤버면 해당 프로젝트 시설물 조회
-- 3. 프로젝트 멤버가 아니면 본인이 생성한 시설물만 조회
```

### 시설물 수정/삭제
```sql
-- Super User면 모든 시설물 수정/삭제 가능
-- PM/OWNER면 해당 프로젝트의 모든 시설물 수정/삭제 가능
-- MEMBER면 본인이 생성한 시설물만 수정/삭제 가능
```
